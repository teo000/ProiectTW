1. Introduction

1.1 Purpose

This document should highlight the software requirements for the Web Technologies course project entitled Crisis Containemnt Service.

This product is intended to be used by both the general public and authorities in order to receive real-time information regarding earthquakes, floodings and fires and also provide escape routes and social media feedback.

1.2 Product Scope

Crisis-Containment-Service is a real-time crisis management software product that aims to provide its users with data regarding calamities (earthquakes, floods, fires) based on their location but not limited to. This product should enrich quality of life by letting its users know exactly what is going on in the world and how to avoid calamities. Authorities can also improve their work using statistics provided by the software product.

2. Overall Description

2.1 Product Functions

The web application is expected to perform the following functionalities:

  • Provide access to a real-time map on the Overview panel that displays which calamities are active.
  • Provide feedback based on the user's location.
  • Give access to Log cards that provide additional information such as the number of affected people, the type of the calamity and a map snipped detailing the location.
  • A section page that previews a Twitter bot that alerts the population through social media.
  • Additionally, messages will be sent to the population in affected areas, providing them with information regarding escape routes.
  • The authorities will have access to statistics and metrics.
  • Login and registration forms. Registration will only be available to authorities that have an e-mail address aproved by the administrator.

2.4 Usage Manual

  1. Overview

    Upon accessing the platform, you will be welcomed by the Overview panel, containing the interactive map, a log section and another that displays the tweets. The interactive map provides the user with the possibility of clicking on pop-ups cards that connects with the logs. The logs update in real-time, allowing the user to scroll through them and click on them, prompting a modal card containing additional information.

  2. Login/Register

    Registration is available only to employees of guvernmental agencines for crisis management. Upon logging in, you can have access to the Dashboard.

  3. Dashboard

    The dashboard panel provides the users with statistics and metrics regarding calamities.

4. System Features

4.1 Registration

The registration feature provides it's users to registrate to the platform using an valid e-Mail. Valid e-mail are e-mail which are accepted and inserted in the database by the administrator.

4.2 Authentication

This is a basic authentication form that checks the validity of the username and the password and redirects the user to the Overview panel. It also unlocks the possibility of viewing the dashboard panel

4.3 Overview Visualisation

4.3.1 Interactive map

The interactive map provides the following functionalities:

  • Zoom-in and Zoom-out
  • Visualize cards that pop-up in real time

4.3.2 Real-time log cards

Log cards are used to display further information regarding the calamity that took place. There are 3 types of calamities:

  1. Earthquakes
  2. Floods
  3. Fires

The log cards provide the following functionalities:

  • Extends cards on-click into modal for more information
  • Scrolling through a bunch of cards. This list updates dinamically.
  • The modal displays a small map to the exact location of the calamity.

4.4 Dashboard visualisation

The dashboard provides information through metrics and statistics to the user. In this case the user is an employee of a guvernmental agency.
